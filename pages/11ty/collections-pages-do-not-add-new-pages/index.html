IF you made a page in 11ty to list all pages that have the same tag, you definitly faced a problem where the new pages do not being added to the collection. That happens because of the cache.

To solve this problem we have two methods :

  • using the --clean flag:

npx @11ty/eleventy --clean

You can add this command to the package.json to be under the scripts so you can run it like this :

npm run clean

  • Using the configuration file .eleventy.js :

This will force it to clean the cache every single time you build the project, This means it will defeat the purpose of having a cache, as your collection would be rebuilt from scratch on every run, which can drastically slow down your build times

To do it write in .eleventy.js :

eleventy.cache.delete(collections:my-custom-collection)

replace my-custom-collection with your collection name.
